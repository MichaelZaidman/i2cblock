
# Overview

The **i2cperf** is a Linux utility to test the large I2C IO chunks' capabilities
and benchmark performance of the I2C controllers and Linux kernel I2C bus
drivers when used with I2C EEPROM chips.

It was originally written to save me the hassle of utilizing the Linux standard
i2c tools to debug and test the hid-ft260 Linux kernel driver [[1]](#References)
for the FTDI FT260 chip [[2]](#References).

This bash script leverages the Linux I2C tools like i2cget, i2cset, i2ctransfer,
and i2cdump, simplifying their configuration to issue predefined I2C IO patterns
allowing performance benchmarking of any I2C controller and Linux kernel I2C bus
driver by measuring payload data rate and efficiency of the IO operations size
at different workloads.

```
 Usage: i2cperf [-h] [-v] -S] [-f MODE] [-d MODE] [-o SIZE] [-s SIZE] [-r FIRST-LAST] BUS ADDRESS

         BUS                    I2C bus master device number
         ADDRESS                EEPROM device address (0x50 - 0x57)

         -h, --help             Show this help message and exit
         -v, --verbose          More verbose output
         -S, --stat             Print statistics

         -d, --dumpmode MODE    Read data mode:
                                1 - Read block via i2cdump byte by byte
                                2 - Read block via i2ctransfer by chunks

         -f, --fillmode MODE    Write data mode:
                                1 - Fill block with zeros via i2ctransfer
                                2 - Fill block with increment via i2ctransfer by chunks
                                3 - Fill block with increment via i2cset byte by byte
                                4 - Fill block with pseudo random via i2ctransfer by chunks

         -r, --range FIRST-LAST Address range to operate
         -s, --xfersize SIZE    Data chunk size in bytes

         -o, --offsize SIZE     Offset (addr in chip addr space) size:
                                1 - One byte offset like in 24C01 - 24C16 EEPROMs
                                2 - Two bytes offest, for larger than 256B EEPROMs
```

## Prerequisites
The **i2cperf** relies on i2ctransfer, added to the [i2c-tools](https://git.kernel.org/pub/scm/utils/i2c-tools/i2c-tools.git)
since v4.0 (2017-10-30).

The i2c-tool usually comes with every Linux distribution and likely will
already be of the needed version.

We can check it by running the `i2ctransfer` command. If it does not exist,
then the i2c-tools package is not installed, or it is of outdated version.

## Tested on
1. Ubuntu 16.04 (Linux 4.15.0-142), MS-7916 PC (i7-4790K, 16G RAM)
2. Ubuntu 20.04 (Linux 5.13.0-30), VirtualBox VM on Win10 HP EliteBook 840 G6 (i5-8265U, 16G RAM)

## Tested with
1. EEPROMS: 24c02, 24c32, 24c512
2. UMFT260EV1A EVB [[3]](#References) and hid-ft260 kernel driver [[1]](#References)
3. CH341A Pro black board [[4]](#References) and 2c-ch341-usb kernel driver [[5]](#References)
   It has issues with voltage as mentioned in [[4]](#References). The clock is
   also far from being accurate. The driver sets 100KHz, but I measured ~350KHz.
4. CH341A green board (077-53900) [[4]](#References) and 2c-ch341-usb kernel driver [[5]](#References)
   The board is slightly overclocked - about 120KHz when configured to 100KHz.


# Usage examples

## Setup: 24c512 EEPROM, CH341A green board (077-53900) [[4]](#References), i2c-ch341-usb kernel driver [[5]](#References), PC (i7-4790K)

### Write 1KB data block with increment via i2cset byte by byte
```
$ sudo ./i2cperf -f 3 -o 2 -r 0-0x3ff 13 0x50 -S

  Fill block with increment via i2cset byte by byte
  -------------------------------------------------------------------
  data rate(bps)  efficiency(%)  data size(B)  total IOs   IO size(B)
  -------------------------------------------------------------------
  7059            21             1024          1024        1
```
